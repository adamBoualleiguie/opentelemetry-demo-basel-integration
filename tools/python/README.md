# Python lockfiles for Bazel (M3 BZ-060 / BZ-061)

Hermetic third-party deps use **`rules_python`** `pip.parse` with lockfiles generated by **`pip-compile`** (from **`pip-tools`**).

## Refresh after editing `requirements.in`

```bash
python3 -m venv .venv-pip
.venv-pip/bin/pip install pip-tools
.venv-pip/bin/pip-compile tools/python/requirements.in -o tools/python/requirements_lock.txt
.venv-pip/bin/pip-compile tools/python/requirements_loadgen.in -o tools/python/requirements_loadgen_lock.txt
```

Use **Python 3.12** to match `MODULE.bazel` `python.toolchain(python_version = "3.12")` and the load-generator Dockerfile base.

## Layout

| File | Consumers |
|------|-----------|
| `requirements.in` / `requirements_lock.txt` | `recommendation`, `product-reviews`, `llm` (`@pypi`) |
| `requirements_loadgen.in` / `requirements_loadgen_lock.txt` | `load-generator` (`@pypi_loadgen`) — Locust + Playwright |

## Proto stubs (`//pb:demo_py_grpc`)

`pb/python/demo_pb2*.py` are copies of the gRPC Python output for `pb/demo.proto` (same as `src/recommendation/`). Regenerate with the repo’s protobuf workflow (`make`/Docker) and copy into `pb/python/` when RPC definitions change.
